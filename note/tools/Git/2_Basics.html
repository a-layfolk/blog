# 基础

# 获取 Git 仓库

两种方式获取 Git 仓库：

  • 将本地目录转换为 Git 仓库
  • 从其他服务器克隆一个已存在的 Git 仓库

# 将本地目录转换为 Git 仓库

$ git init # 生成一个 .git 的子目录

这是项目中的文件还没有被跟踪， 需要执行 git add 和 git commit 命令：

$ git add README.md
$ git commit -m "initial project version"

# 克隆现有仓库

$ git clone <url> # 例如 git clone https://github.com/libgt2/libgit2

$ git clone <url> <alias> # 克隆远程仓库， 自定义本地名字

# 记录每次更新到仓库

工作目录下，文件只有已跟踪和未跟踪两种状态。

# 检查当前文件状态

$ git status
On branch master
Your branch is up-to-date with 'origin/master'.
# 当前目录没有出现任何处于未跟踪状态的新文件
nothing to commit, working directory clean

$ git status
On branch master
Your branch is up-to-date with 'origin/master'.
# 表明 README 没有被跟踪
Untracked files:
  (use "git add <file>..." to include in what will be committed)

    README

nothing added to commit but untracked files present (use "git add" to track)

# 跟踪新文件

$ git add <files>

# 暂存已修改文件

假如我们修改已被跟踪的文件 README.md， 运行 git status:

$ git status
On branch master
Your branch is up-to-date with 'origin/master'.

# 跟踪文件发生了变化， 但还没有放到暂存区
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

    modified:   README.md

为了暂存这次更新， 我们也要使用 git add 命令。

TIP

git add 是一个多功能命令， 可以用它开始跟踪新文件， 可以把已跟踪文件放到暂存区， 可以合并时把带有冲突的文件标记为已解决状态。

将命令理解为 精确地将内容添加到下一次提交中 而不是 "将一个文件添加到项目中" 更合适

# 状态简览

# 使用 git status -s 或者 git status --short 可以查看简洁的更改
$ git status -s
# REAME 已修改但未暂存
 M README
# Rakcefile 文件已修改， 暂存后又作了修改， 该文件修改的部分既有修改的也有未修改的
MM Rakefile
# 新添加到暂存区的有 A 标记， lib/git 已修改且已暂存
A lib/git.rb
# 修改过的文件前面有 M 标记
M lib/simplegit.rb
# 新添加的未跟踪的文件前面有 ?? 标记
?? LICENS.txt

# 忽略文件

一些文件不需要纳入 Git 管理， 也不希望它们总出现在未跟踪的列表中。 我们可以创建 .gitignore 文件列出要忽略的文件的模式。

格式规范如下：

  • 所有的空行或者以 # 开头的行都会被 Git 忽略（即 # 可以写注释）
  • 可以使用标准的 glob 模式匹配， 它会递归地应用在整个工作区中

    TIP

    所谓的 glob 模式是指 shell 所使用的简化了的正则表达式：

    • * 匹配零个或多个任意字符
    • [abc] 匹配任何一个列在方括号中的字符
    • ? 只匹配一个任意字符
    • [0-9] 匹配所有 0 到 9 的数字
    • ** 表示匹配任意中甲目录， 比如 a/**/z 可以匹配 a/z， a/b/z 等
  • 匹配模式可以以 / 开头防止递归
  • 匹配模式可以以 / 结尾指定目录
  • 要忽略指定模式以外的文件或目录， 可以在模式前加上 ! 取反

详细例子可以查看 Git 书籍 (opens new window)

# 查看已暂存和未暂存的修改

如果认为 git status 输出过于简略， 可以使用 git diff

# 比较修改之后还没有暂存起来的内容
$ git diff

# 比对已暂存文件和最后一次提交的文件差异
$ git diff --staged
# --staged 和 --cached 是同义词
$ git diff --cached

# 提交更新

# 启动你选择的文本编辑器输入提交说明
$ git commit

# 将提交信息与命令放在一行
$ git commit -m "something changed"

# 跳过使用暂存区域

$ git commit -a -m "something changed"

# 移除文件

# 从跟踪文件中删除， 连带着工作目录中一起删除
$ git rm <files>

# 删除已经修改过或已经放到暂存区的文件， 必须使用 -f（force 的首字母）
$ git rm -f <files>

# 将文件从暂存区删除， 但文件仍保留在当前工作目录
# 不小心将日志文件或一大堆编译文件假如暂存区时， 这一方法很有用
$ git rm --cached <files>
# git rm 可以使用 glob 模式
$ git rm log/\*.log

# 重命名文件

$ git mv file_from file_to

此命令相当于执行了三条命令：

$ mv file_from file_to
$ git rm file_from
$ git add file_to

# 查看提交历史

$ git log

常用的 git log 选项：

  • -p
    # -p 或 --patch 显示每次提交引入的差异（按 补丁 的格式输出）， -2 显示最近两次提交
$ git log -p -2
  • --stat
    # 查看每次提交的简略信息
$ git log --stat

更多的选项我觉得用不到， 以后需要时再查看：Git 数据 (opens new window)

# 撤销操作

有时提交完发现漏掉了几个文件， 或者提交信息写错了， 此时可以用 --amend 重新提交

# 最终第二次提交将代替第一次提交
$ git commit --amend

# 取消暂存的文件

$ git reset HEAD <file>

# 撤销对文件的修改

$ git checkout --<file>

# 远程仓库的使用

TIP

远程仓库可以再你的本地主机上

# 查看远程仓库

$ git remote
# 至少应该看到 origin —— 这是 Git 给你仓库服务器的默认名字
origin

$ git remote -v
# 显示简写和对应的 URL
origin  git@github.com:a-layfolk/anonymous-chat.git (fetch)
origin  git@github.com:a-layfolk/anonymous-chat.git (push)

# 添加远程仓库

# 添加一个新的 Git 仓库同时指定一个方便的简写
$ git remote add <shortname> <url>

# 从远程仓库中抓取与拉取

$ git fetch <remote>

TIP

git fetch 命令只会将数据下载到本地仓库， 而不会自动合并或修改你当前的工作。 你必须手动将其合并入你的工作

# 推送到远程仓库

$ git push <remote> <branch>

# 查看某个远程仓库

$ git remote show

# 远程仓库的重命名与移除

  • 重命名

    $ git remote rename from_name to_name

  • 移除

    这种方式会将远程跟踪分支和配置信息一起删除

    # 也可以使用 remove 的简写 rm
$ git remote remove <remote>

# 创建标签

Git 支持两种标签：

  • 轻量标签（lightweight）

    轻量标签很像一个不会改变的分支， 它只是某个特定提交的引用 （这句话意思不是非常懂）

  • 附注标签（annotated）

    是存储在 Git 数据库中的一个完整对象， 可以被校验， 其中包含打标签者的名字， 电子邮箱地址， 时间

TIP

通常建议创建附注标签

# 附注标签

-a 创建附注标签， -m 指定存储在标签中的信息

$ git tag -a v1.4 -m "my version 1.4"

git show 命令可以看到标签信息和提交信息

$ git show v1.4

# 轻量标签

# 不需要使用 -a -m 选项， 只需要标签名字
$ git tag v1.4

# 后期打标签

这里不是很清除， Git 书籍 (opens new window)

# 共享标签

默认情况下， git push 命令不会上传标签， 你需要显示地推送标签到共享服务器：

$ git push origin <tagname>

# 一次性推送很多标签
$ git push origin --tags

# 删除标签

# 删除本地标签
$ git tag -d <tagname>

# 删除远程仓库标签
# 意思是将冒号前面的空值推送到远程标签名， 从而删除
$ git push <remote> :refs/tags/<tagname>

# 更直观的删除远程标签的方式
$ git push origin --delete <tagname>

# Git 别名

可以通过 git config 文件来轻松地为每一个命令设置一个别名：

# 将 checkout 别名设置为 co
$ git config --global alias.co checkout
