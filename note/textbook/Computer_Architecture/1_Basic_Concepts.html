TIP

各机器级的实现主要靠翻译或解释，或者是这两者的集合。

翻译（Translation）是用转换程序将高一级的机器语言变为等效的低一级机器语言。然后再实现。

解释（Interpretation）使用低一级机器语言逐条解释来实现高一级机器语言。

# 计算机系统结构、组成与实现

  • 计算机体系结构是指机器语言程序员所看到的传统机的属性，包括指令集、数据类型、存储器寻址计数、I/O 机理等，大都术语抽象的属性。
  • 计算机组成是指如何实现计算机系统结构所体现的属性，它包含了许多对程序员来说是透明的（即程序员不知道的）硬件细节。
  • 计算机实现是指计算机组成的物理实现，包括处理机、内存的物理结构，器件的急程度和速度等。它主要着眼于器件技术和微组装技术，其中器件技术在实现技术中起着主导作用。
  • 相同系统结构的计算机，可以因速度不同而采用不同的组成。同样，一种组成可以采用多种不同的实现方法。

WARNING

日常生活的“透明”是要公开，让大家看得到，而计算机中的“透明 ” ， 则是指看不到的意思， 实际上就是指那些不属于自己管的部分。

# 软硬件取舍与计算机系统的设计思路

确定软硬件功能分配的准则：

  • 在现有硬件条件下，系统要有高的性能价格比，主要从实现费用、速度和其他性能要求来综合考虑
  • 考虑到准备采用和可能采用的组成技术，使它尽可能不要过多或不合理地限制各种组成、实现技术的采用
  • 不能仅从“硬”的角度支持应用组成技术和器件技术，还应从“软”的角度支持操作系统的实现、高级程序语言的设计

计算机系统的设计思路：

  • 由上往下设计。完全是面向特定的应用优化设计的，适合专用机的设计，不适合通用机。
  • 由下往上设计。很少被采用，会造成软件和硬件脱机。（因为如今硬件飞速发展，软件技术发展相对缓慢）
  • 由中间开始设计。“中间”指操作系统级与传统机器级之间。

# 对系统结构的影响因素

软件对系统结构的影响

软件对系统结构的影响最重要的是软件的可移植性问题。即统一软件可以应用于不同环境，其差别只是执行时间不同。

实现软件可移植性的常用方法有三种：统一高级语言，采用系列机，模拟与仿真。

器件和应用对系统结构的影响

  • 器件的发展是推动系统结构和组成技术前进的关键因素，这是因为器件是组成计算机系统最基本的单位。
  • 应用对系统结构的发展有着重要的影响。不同的应用对计算机系统结构的设计提出了不同的要求。因此，应用需求是促使计算机系统发展最根本的动力。

# 系统结构中的并行性

并行性，是指计算机系统在同一时刻或者同一时间间隔内进行多种运算或操作。

从执行程序的角度分级 从处理数据的角度分级 从计算机信息加工的角度分级
1. 指令内部并行——一条指令内部各个微操作之间的并行
2. 指令级并行——多条指令的并行执行
3. 线程级并行——并行执行两个或两个以上的线程
4. 任务或进程间并行——多个任务或程序段的并行执行
5. 作业或程序间并行——多个作业或多道程序的并行		 1. 位串字串——同时只对一个字的一位进行处理，这就是串行
2. 位并字串——同时对一个字的全部位进行处理，不同字之间是串行
3. 位片串字并——同时对许多字的同一位（称为位片）进行处理
4. 全并行——同时对许多字的全部或部分位进行处理
 1. 存储器操作并行
2. 处理器操作步骤并行
3. 处理器操作并行
4. 指令、任务、作业并行

计算机系统中提高并发性的措施可以归纳为以下三条途径：

  • 时间重叠
  • 资源重复
  • 资源共享
