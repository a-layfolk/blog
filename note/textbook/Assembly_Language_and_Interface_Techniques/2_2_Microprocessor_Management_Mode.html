# 微处理器管理模式（中）

# 内存管理

CPU 通过对内部存储器的管理实现任务切换和资源调配。早期的 16 位 CPU 通过分段方法实现对内存的管理，32 位 CPU 采用分段和分页结合的方式实现对内存的管理。

# 实模式下分段管理

分段管理的原因

8086/8088 寄存器的大小是 16 位（范围是 64 KB），物理地址的大小是 20 位（范围是 1 MB）。这就造成一个矛盾，单独使用一个 16 位数不能表示或确定某一内存单元的物理地址。

解决办法

Intel 引入段的概念：

  • 段的大小可以变化
  • 16 位 CPU 中最大的段为 2^16 = 64 KB，最小的段没有限制，因此可以定义 1B 的段
  • 段的起始地址（简称段基址、段首址）可以在任何 16 的倍数上，这种段起始边界通常称为节或小段
  • 段寄存器中存放 20 位段基址的高 16 位

引入分段管理后，一个内存单元地址要用段基地址和偏移量两个逻辑地址来描述，表示为“段基址：偏移量”。

TIP

因此，程序中给出的内存单元地址只是逻辑地址，而不是物理地址，真正的物理地址要通过 CPU 部件自动计算得到。

存储器采用分段管理后，其物理地址的计算方法为：10H × 段基址 + 偏移量（其中 H 表示是十六进制数）

# 保护模式下分段管理

保护模式下，逻辑地址同样由 “段基址：偏移量”的格式给出。不过用来存储段基址的段寄存器不再表示段的起始位置，而是用来表示段选择符，偏移量在保护模式下也是由 32 为的寄存器或存储器寻址方式给出的。

TIP

保护模式下，该地址表示形式称为虚拟地址，对应的地址空间称为虚拟地址空间。

至于详细如何计算，可以看书 45 页的例子。

1. 段描述符

段描述符位于 GDT 或 LDT 中，占 8 字节，包括段基址（32 位），限长（20 位）和属性（4 位）。

属性位中有一个 G（Granularity），叫做粒度。G = 1 时，限长以页位单位；G = 0 时，限长以字节为单位。

限长位在描述符中一共占 20 位。G = 1 时，线程的内容加上 1 就是段所占的页数，1 页的大小为 2 ^12 = 4 KB；G = 0 时，段限长以字节为单位，限长的内容 + 1 就是段所占的字节数。

2. 段的属性

段的属性是由段选择符来决定的，对于同样一个线性地址范围，不同的选择符有不同的属性。

3. 段描述符高速缓存

内存读写时，CPU 要计算线性地址，需要取得段描述符中的基地址再加上偏移量。接下来还需要根据段描述符的属性来操作。这样很耗时。

于是 CPU 在内部设置了段描述符高速缓存。

段描述符高速缓存总是与 CS、DS、ES、SS、FS、GS 段寄存器和描述符当前值保持一致，只有段寄存器的值发生改变时，才需要到 GDT 或 LDT 中装入段描述符。

4. 段式地址转换

CPU 的分段部件完成将虚拟地址转换为线性地址。

CPU 的分页部件将线性地址转换为物理地址。如果禁止 CPU 的分页功能，线性地址就直接作为物理地址。

# 页式内存管理

分页管理是在分段管理机制的基础上工作的。

使用分页机制的好处在于它可以把每个活动任务当前所必须的少量页面放在内存中，而不必将整个段调入内存。

1. 分页

由于线性地址是 32 位，所以线性地址空间为 2^32 = 4GB。

管理模式 特点
分段 段的长度不固定，段与段之间允许重叠
分页 页的长度固定，页与页之间没有重叠

2. 线性地址到物理地址的映射过程

TIP

每一个线性页面只能映射到一个物理页面上，但是多个线性页面可以映射到同一个物理页面上。

分页机制就是一种将线性地址的页面映射到物理地址页面的手段，它用到了两个表，页目录表和页表。

3. 片内转换检测缓冲器

页式内存管理中，每次内存操作都需要将线性地址转换为物理地址，每次都需要两次访问主存，分别访问页目录表和页表来取得页表描述符和页描述符，时间开销大。

于是 CPU 内置了片内转换检测缓冲器 TLB（Translation Look-aside Buffer），其中保存了 32 个页描述符，它们都是最近使用过的。

4. 页表项

页表项就是在分页转换时用到的页表描述符和页描述符，它们都是 32 位的，其格式相同。

  • 页目录表、页表和页面的基地址的低 12 位全部为 0
  • 页表描述符的高 20 位就是页表基地址的高 20 位
  • 页描述符的高 20 位就是物理页面基地址的高 20 位

页表项的低 12 位提供保护功能和统计功能，具体可以看书 54 页。
