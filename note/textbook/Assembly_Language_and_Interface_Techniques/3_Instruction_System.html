# 汇编语言与接口技术

一条指令通常由操作码域和操作数域两部分组成，操作码域指示计算机要执行的操作，操作数域则提供与操作数或操作数地址有关的信息。

# 数据寻址方式

以 MOV 为例子，MOV 指令的汇编格式为

MOV 目标, 源			;把源操作数传送给目标
  • 目标和源是操作数，中间用逗号隔开
  • 注释内容从 ";" 开始

# CPU 操作数寻址

1. 立即寻址方式

操作数直接包含再指令中，紧跟在操作数之后的寻址方式称为立即寻址方式，把该操作数简称为立即数。例如：

MOV BL, 9			;执行结果（BL）= 9
  • 立即数不能作为目标操作数
  • 源操作数和目标操作数位数要一致

2. 寄存器寻址方式

操作数直接包含在寄存器中，由指令指定寄存器的寻址方式。例如：

MOV BX, AX

TIP

一般通用寄存器可以出现在寄存器寻址方式中。

而一般 MOV EIP OFH 类指令不合法，因为 EIP 比较特殊。

# 存储器操作数寻址

参与运算的操作数来自存储器（RAM）

1. 直接寻址方式

操作数的有效地址 Effective Address（EA）直接包含在指令中。

TIP

EA 只是操作数地址的偏移量部分。其物理地址还需要把有效地址和段基址相加得到。

  • 80x86 实模式下，段基址 = 段寄存器内容 x 16
  • 保护模式，段寄存器的描述符索引 ---从描述符表中得到描述符--> 从描述符中得到段基址
MOV EAX,[00404011H]			;把 00404011H 单元中的内容送给 EAX

MOV EAX,VAR					;VAR 是一个内存变量名，代表一个内存单元的符号地址
				;实际上，在汇编语言源程序中所看到的直接寻址方式通常都是用符号表示的

MOV EBX,ES:MEM				;该指定源操作数的有效地址时用变量名表示的 MEM

2. 寄存器间接寻址方式

  • 操作数的有效地址在寄存器
  • 操作数本身在存储器
  • 对于 16 位寻址，这个寄存器只能是基址寄存器 BX、BP 或变量寄存器 SI、DI
  • 对于 32 位寻址，允许使用任何 32 位通用寄存器
MOV AL,[BX]			;BX 用 [] 括起来，它表示 BX 寄存器中是有效地址而不是操作数
