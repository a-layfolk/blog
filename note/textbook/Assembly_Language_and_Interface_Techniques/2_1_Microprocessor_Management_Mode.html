# 微处理器管理模式（上）

# 微处理器的基本结构

80x86 系列成员都采用并行处理技术，即微处理器中有多个处理单元在同一时间段内协同工作，每个处理单元负责一项工作，并行处理得越多，则微处理器的性能就越高。
8086/8088 总线接口单元（BIU） 执行单元（EU）
80286 总线单元 指令单元 执行单元 地址单元
80386 总线接口单元（BIU） 中央处理单元（CPU） 存储器管理单元（MMU）

  • 总线接口单元。通过数据总线、地址总线、控制总线来与外部环境联系。包括从存储器中预取指令、读写数据，从 I/O 端口读写数据及其他的控制功能。
  • 存储器管理单元。由分段部件与分页部件组成，提供存储器管理和保护服务，实现从逻辑地址到物理地址的转换，既支持短时存储管理、页式存储管理，也支持段页式存储管理。
  • 中央处理单元。由指令部件和执行部件组成。
    • 指令部件。
      • 指令预取单元，负责从存储器中取出指令，放到 FIFO 的指令预取队列。它的优先级低于数据传送等总线操作，绝大部分时利用总线空闲时间预取指令。
      • 指令译码单元，从指令预取队列中取出指令，进行译码，并将译码后的可执行指令放入到已译码指令队列中。
    • 执行部件。负责具体指令的执行操作，从已译码队列中取指令执行。如果是算术、逻辑或者移位运算，交给 ALU 处理；若指令执行时需要段或者页单元产生操作数地址，则交给分段或者分页单元进行处理。

# CPU 工作模式

从 80386 开始，32 位 CPU 具有 3 中运行模式：实模式、保护模式和虚拟 8086 模式。运行模式及其转换关系如下图所示：

# 实模式

CPU被复位（加电）时，自动进入实模式。

  • CPU 相当于高性能的 8086
  • 使用 1 MB 地址空间及 16 位的 ”段首址：偏移“的地址格式
  • CPU 其他功能没有得到发挥
  • 不支持硬件上的多任务切换
  • 不能对内存进行分页管理
  • 所有段都可以读、写和执行
  • 不支持特权级，即所有程序均可执行特权指令

TIP

DOS 操作系统运行于实模式下，而 Windows、Linux 操作系统运行于保护模式下。

# 保护模式

通过修改控制寄存器 CR0 的控制位 PE（位 0）来实现从实模式切换到保护模式，在这之前，还需要建立保护模式必须的一些数据表，如全局描述符表 GDT 和中断描述符表 IDT 等。

  • 支持内存分页机制
  • 提供段式和页式内存管理功能
  • 协作操作系统搞笑地实现虚拟内存、支持多任务和特权级等
  • 物理寻址空间高达 4GB（80386/80486）或 64 GB（Pentium 及以上 CPU）
  • 执行 JMP/CALL/IRET 等指令就可以实现任务切换
  • 在保护模式下，CPU 由 4 个特权级
图片 特权级
操作系统在最高特权级上（RIng 0），存储器的管理（包括虚拟存储器）、任务间的通信、设备的 I/O 等服务均属于这一层次，称为核心态
特权级1，依赖于核心态，以便典型地用于系统服务，包括文件共享、显示管理及数据通信等
特权级2，专用于用户拓展
特权级3，运行应用程序

TIP

理论上，处于特权级 3 的应用程序之间及它们和操作系统之间相互隔离，应用程序不能直接修改操作系统的程序和数据，也不能对其他应用程序的操作造成影响，形成保护。

特权级 1 和 特权级 2 目前在 Windows 和 Linux 操作系统中没有被使用。

# 虚拟 8086 模式

V86 模式时为了在 Windows、Linux 系统中执行 DOS 程序而设计的，它是一种经过”修改“的保护模式。

  • 采用和 8086 一样的寻址方式，即用段寄存器乘以 16 作为基址，再加上便宜地址形成物理地址。寻址空间为 1 MB。
  • 以任务形式再保护模式下执行，每个任务都有自己的任务状态段，各个 V86 任务所拥有的 1 MB 地址空间是相互独立的。
  • CPU 可以同时支持多个真正的 80386 任务和多个 V86 模式任务。

DOS 程序中有相当一部分指令在保护模式下属于特权指令，如屏蔽指令 CLI、中断指令 INT、中断返回指令 IRET 等。V86 采用模拟的方法来完成这些指令：

指令在 V86 模式运行时，引起保护异常。

  • 如果是中断指令，则从 V86 任务的中断向量表取出中断处理程序的入口地址，并将控制转移过去。
  • 如果是危及操作系统的指令，如 CLI 等，则简单地忽略这些指令，继续执行下一条指令。

# 64 位 CPU 的工作模式

以 Intel 公司的 EM64T 为例：

EM64T 全名拓展 64 位内存技术（Extended Memory 64 Technology）。是在 IA-32 指令集的基础上进行拓展的。设定了两种模式：

  • IA-32 就是传统的模式（Legacy Mode）。本质上就是把 CPU 中所有为 64 位 CPU 计算而新增的运算机制都频闭起来，现有的 x86 程序无需做任何的改变就能运行。

  • IA-32e

    • 兼容模式

      目前是在 64 位操作系统下最常见的模式，就像传统的保护模式一样。允许 64 位操作系统运行基于 32 位和 16 位代码的程序。

    • 64 位模式

      是 EM64T 技术中最为高效的模式，需要纯 64 位环境的支持，包括 64 位操作系统和 64 位应用程序。

# 寄存器

寄存器可分为：

  • 程序可见寄存器组

    是指汇编语言程序设计人员在应用程序设计期间可以使用的寄存器

  • 程序不可见寄存器组

    是指在应用程序设计时不能直接使用，但在系统程序或系统运行程序期间可能要用到的寄存器

TIP

80386 以上 CPU 才包含程序不可见寄存器组。

# 程序可见寄存器组

Intel 80x86 系列程序可见寄存器组包括：

  • 通用寄存器
  • 段寄存器
  • 控制寄存器

1. 通用寄存器

可以被程序访问，常用于传送和暂存数据，也可参与算术逻辑运算并保存运算结果。除此之外还各自具有一些特殊功能。具体见书 36 页表 2-1 和 表 2-2。

2.段寄存器

段寄存器用来确定一个存储段在内存中的起始字段。

  • 80286 以前 CPU 有 4 个段寄存器：

    分别为：

    • 代码段寄存器 CS（Code Segment）
    • 数据段寄存器 DS（Data Segment）
    • 堆栈段寄存器 SS （Stack Segment）
    • 附加数据段寄存器 ES（Extra Segment）

  • 80836 CPU 开始，增加了 FS 和 GS 两个段寄存器

段寄存器 作用
CS 指定当前代码段的及地址，代码段中存放正在运行的程序
DS 指令当前运行程序所使用的数据段
SS 指定当前堆栈段的段基址，堆栈段时在内存开辟的一块特殊区域，SP（32 位下为 ESP ）指向栈顶，SS 指向堆栈段基址
ES 指定当前运行程序所使用的附加数据段
FS、GS 只对 80386 以上机器有效，用于指定当前运行程序的另外两个存放数据的存储段

3. 专用寄存器

以 Intel 8086~Core2 为例，专用型寄存器包括：

  • 指令指针寄存器

    指向程序的下一条指令。

    • 8086/8088、80286 或工作在实模式下，16 位 IP
    • 80386 及更高型号的微处理器，使用 32 位 EIP
    • 64 位模式中，RIP 包含 40 位地址址先，可以寻 1 TB 平坦模式地址空间

  • 堆栈指针寄存器

    指向栈顶单元。

    • 16 位，SP
    • 32 位，ESP
    • 64 位，RSP

  • 标志寄存器

    指示微处理器状态并控制它的操作。

    • 8086/8088、80286 下，16 位 FLAG 寄存器
    • 80386 及更高微处理器使用 32 位 EFLAG 寄存器
    • 64 位版本中的 RFLAG 包含 EFLAG 和 FLAG 寄存器

    至于 FLAG 标志寄存器有效位的含义，可以看书 38 页。

# 保护模式下的寄存器

以 80386 为例，新增了一系列寄存器用于支持 CPU 工作于保护模式下。

1. 控制寄存器

80386 中有四个系统控制器：CR0~CR3。

CR0 的低 5 位是系统控制标志，被称为机器状态字，分页机制中用到 CR3、CR2 和 CR0

更多信息看书上 40 页

2. 测试和测试寄存器

80386 提供 8 个 32 位的测试寄存器和 8 个 32 位的测试寄存器。

  • 调试寄存器。为调试软件的错误提供硬件支持，其中 DR0 ~ DR# 保存线性断点地址供分析；DR5 为保留位；DR6 为调试状态器；DR7 为调试控制寄存器。具体看书 41 页。
  • 测试寄存器 TR0~TR5 保留，用户可见的只有 TR6、TR7。它们用于转换检测缓冲区（Translation Look-aside Buffer）中测试相关存储器。

3. 全局描述符表寄存器

  • 全局存储器是一种共享系统资源，该存储器可以被所有任务访问。
  • 全局描述符表（GDT）是用来定义全局存储器空间的一种机制，用段描述符来描述全局存储器中的段，每个 GDT 最多含有 2^13（即 64 KB） 个描述符。
  • GDT 可以存储在内存的任何位置。全局描述符表寄存器（GDTR）可以给出 GDT 的位置。

  • GDTR 是 48 位寄存器
  • GDTR 最低 16 位是线长，给出 GDT 字节的大小（其值比 GDT 的长度少 1）
  • GDTR 高 32 位是基址，指出 GDT 在物理存储器中存放的基地址

4. 中断描述符寄存器

  • 中断描述符表 IDT 保存中断门描述符。每个门描述符包含 8 字节。IDT 最多包含 256 个门描述符，因为 CPU 最多支持 256 个中断。
  • 中断门指向的是中断服务器的入口。
  • IDTR 可以给出 IDT 的位置

  • IDTR 是 48 位寄存器
  • IDTR 最低 16 位是限长，给出 IDT 的字节大小（其值比 IDT 的长度少 1）
  • IDTR 高 32 位是线性地址，经过分页部件转换后位物理地址。

TIP

GDTR 和 IDTR 的值必须在进入保护模式之前装入。

在实模式下执行 LGDT 和 LIDT 指令装入 GDTR 和 IDTR。

5. 局部描述符表寄存器

  • 保护模式下提供了多任务的环境，除了 GDT 外，每个任务都有一个局部描述符表 LDT。

  • LDTR 并不直接指出 LDT 的位置和大小。

图片 过程
① 和 ② 由 GDTR 确定了 GDT 表在存储器中的位置和限长。
LDTR 是一个选择符，包含了 LDT 描述符在 GDT 中的索引。
③ 步是依据 LDTR 在 GDT 中取出 LDT 描述符的过程。
在 LDT 描述符中，包含有 LDT 的位置和限长，即 ④ 和 ⑤ 步。

6. 任务寄存器

  • 任务寄存器（TR）在保护模式的任务切换机制中使用。
  • TR 是 16 位的选择符，它选中的是 TSS 描述符。
  • TR 的初值由软件装入，当执行任务切换指令时，TR 的内容自动修改。
  • TSS 描述符放在 GDT 中
图片 过程
① 和 ② 步由 GDTR 确定了 GDT 表在存储器中的位置和限长。
③ 步依据 TR 在 GDT 中取出 TSS 描述符。
在 ④ 和 ⑤ 步中，在 TSS 描述符中取得 TSS 的基址和限长。

7. 段选择符

段寄存器在实模式、V86 模式和保护模式下有不同的内容。

  • 实模式、V86 模式

    用法兼容 16 位 CPU，即段寄存器保存 20 位段基址的高 16 位，低 4 位为 0。

  • 保护模式

    不直接存放段基地址，而是存放一个索引，称为段选择符（Segment Selector）。

    由段选择符从 GDT 或 LDT 中找到 8 字节长的段描述符，从而确定关于这个段的全部描述信息。

图片 描述
① RPL：请求权限级。取值范围为 00 ~ 11，分别代表特权级 0、1、2 和 3
② T1：表指示符。为 0 时从 GDT 中选择描述符，为 1 时从 LDT 中选择描述符
③ Index：索引。指出要访问描述符在段描述符表中的顺序号，Index 占 13 位。
