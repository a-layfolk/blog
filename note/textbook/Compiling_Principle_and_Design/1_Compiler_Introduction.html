# 编译引论

# 定义

定义1.1：将某一种程序设计语言写的程序翻译成等价的另一种语言的程序的程序,称之为编译程序

  • 编译程序（编译器 compiler）或翻译程序（翻译器 translator）。
  • 源程序（source program）
  • 目标程序（target program）
  • 实现编译程序的语言称之为宿主语言

# 编译程序的表示

  • 函数表示，T = C(S)
  • T 型图表示
  • 符号表示

# 编译程序的分类

  • 解释执行
  • 编译执行

从实用角度又可以分为：并行编译器、优化型编译器、交叉型编译器、诊断型编译器和可重定向型编译器。

TIP

交叉编译：由于目标机 B 指令系统与宿主机 A 的指令系统不同，要用运行在宿主机 A 上的某编译程序为另一台机器 B 生成目标代码。

交叉编译，中间 T 型图的任务是把左边 T 型图的 宿主语言 作为源语言，把右边 T 型图的的 宿主语言 作为目标语言。

# 编译过程

  1. 词法分析

    识别出符号串的每个字符，并标注属性。是一种线性分析。

  2. 语法分析

    依据语法规则进行语法检查，并识别出单词序列对应的语法范畴。结果一般为分析树（parse tree）或者语法树（Syntax tree）。

  3. 语义分析

    进行语义检查和处理，翻译为等价的中间代码或者目标代码。

  4. 代码优化

    一般会想办法节省运行时间和存储空间。

  5. 目标代码生成

  • 分析阶段（前端）：重在语言结构分析，完成词法分析、语法分析和语义分析，一般与目标机无关，适用于自动生成。
  • 综合阶段（后端）：一般与目标机有关。
  • 表格管理：在对源程序分析中，需要保存一系列表格用来登记信息，编译过程需要不断建表、查表和填表
  • 错误处理：编译程序应该能帮助程序员找出源程序的错误

定义1.2： 对源程序或源程序的中间形式从头到尾扫描一遍，并做有关的分析加工，生成新的源程序的中间形式或生成目标程序

每一遍可以完成一个或相关几个逻辑阶段的工作，一个阶段的工作也可以分几遍完成。

# 编译程序的构造与实现

构成一个编译程序的三要素是源语言，目标语言，编译方法、技术及工具。

编译程序的生成方式：

  • 移植：将某条机器现有语言的编译程序移植到另一台机器

  • 自展方式，也叫自编译：先对语言核心部分用其他语言（汇编语言和机器语言构造一个小的编译程序），然后利用核心部分构造更多语言成分的编译程序。

  • 自动生成：翻译程序编写系统（Translator Writing System, TWS）

    TWS 的发展先后出现了许多工具

    • 词法分析器自动生成器，例如 Lex 和 Flex
    • 语法分析器程序产生器
    • 语言制导翻译器
    • 代码自动生成器
    • 数据流分析装置
