# 基础类型

# 布尔类型

let flag: boolean = true;

# 数字类型

和 JavaScript 一样， TypeScript 的数字类型都是浮点数。 TypeScript 支持二进制、 八进制、 十进制和十六进制。

let num1: number = 0b1010;   // 二进制
let num2: number = 0o12;     // 八进制
let num3: number = 10;       // 十进制
let num4: number = 0xa;      // 十六进制

# 字符串

TypeScript 的字符串和 JavaScript 的字符串语法基本相同：

let str1: string = "床前明月光";
let str2: string = '疑是地上霜';
console.log(`${str1},${str2}`);
// 打印 床前明月光,疑是地上霜

let num: number = 1000;
let str: string = "Hello";
console.log(str + num);
// 打印 Hello1000

# undefined 和 null

let und: undefined = undefined;
let nul: null = null;

TIP

在非严格模式下， undefined 和 null 都可以赋值给其他类型。

# 数组

// 数组定义方式一
let arr1: number[] = [1, 2, 3, 4];
// 数组定义方式二：泛型写法
let arr2: Array<number> = [100, 200, 300];

# 元组

JavaScript 中数组中各个位置元素类型可以不一样。 TypeScript 提供元组来满足这个要求。

let tuple: [string, number, boolean] = ["Hello", 100.222, true];
console.log(tuple[0].split(""));  
// 打印 ["H", "e", "l", "l", "o"]
console.log(tuple[1].toFixed(1));
// 打印 100.2

# 枚举

enum Color {
  red,
  green,
  blue
};

let color: Color = Color.red;
console.log(color);
// 打印 ，0

# any

any 可以当作任何类型

let arr: any[] = [100, "talk is cheap", true];
console.log(arr);

TIP

当一个数组中要存储多个数据， 个数和类型都不确定时， 可以使用 any 类型。

但使用 any 类型也有缺点 —— 出错后没有报错信息。

# void

void 类型代表不是任何类型， 常用来表示函数没有返回值

function showMsg(): void {
  console.log("this function has no return value");
}
console.log(showMsg());
// undefined
