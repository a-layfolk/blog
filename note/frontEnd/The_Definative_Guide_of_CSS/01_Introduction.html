# 引言

作者个人网站 (opens new window)

1995 年 W3C 发布 CSS，用来解决 HTML 充斥着各种表现标记的问题。

CSS 易于使用，能同时被多个 HTML 页面引用，制定了冲突规则——层叠（cascade）。

CSS 中元素通常有两种形式：替换和非替换

  • 替换元素（replaced element）

    内容并非由文档内容直接显示，比如 img 元素，它有文档之外的图片文件来替换。

    input 元素与之类似。

  • 非替换元素。（nonrepalced element）

    段落、标题、表单元格、列表等几乎所有元素都是非替换元素。

另一种分类是块级元素和行内元素：

  • 块级元素

    生成一个元素框，（默认地）它会填充其父元素的内容区，旁边不能有其他元素。

    它在元素框之前和之后生成了 “分隔符”。

    TIP

    替换元素可以是块级元素，不过通常都不是。

  • 行内元素

    行内元素在一个文本行内生成元素框，而不会打断这行文本。

TIP

在 HTML 中，块级元素不能集成自行内元素，但是在 CSS 中，对于显示角色如何嵌套不存在任何限制。

HTML 引入 CSS 使用 link 标记：

<link rel="stylesheet" type="text/css" href="sheet1.css" media="all" />

link 必须放在 head 元素中，但不能放在其他元素内部（如 title）

  • rel 代表关系（relation），关系为 stylesheet
  • type 总是设置为 text/css
  • href 可以是绝对 URL 也可以是相对 URL
  • media 表示要应用的媒体，除了 all 之外，还有许多其它值，具体请看书 21 页

候选样式表：

将 rel 属性的值设置为 alternate stylesheet，只有当用户选择这个样式表时才会用于文档表现。

还可以为候选样式表指定同样的 title 值，把它们分组在一起，这样一来，用户就可以在品目和打印媒体中为网站选择不同的表现。

style 元素：

<style type="text/css">
</style>

@import 指令：

@import 出现在 style 容器中，它必须放在这里，也就是要放在其他 CSS 规则之前，否则将根本不起作用。

注释:

/* this css comment is serveral lines */

内联样式：

如果只想为单个元素指定一些样式，可以用内联样式。

<p style="color: gray;">this is a paragraph</p>
