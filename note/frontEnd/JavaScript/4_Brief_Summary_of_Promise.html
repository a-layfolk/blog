# Primise 简要总结

谈 Promise 的书籍很多， 身为一个菜鸟， 我对其并不是很理解。 下面总结我认为的 Promise 。

  • Promise 是一个容器， 里面包含着未来才能完成的事件。

  • Promise 对象有三种状态， pending(运行中)， fulfilled(已成功)， rejected(已完成)。 只有异步操作的结果可以改变这三种状态， 让状态从 pending 转变为 fulfilled， 或者从 pending 转变为 rejected， 这种状态的转变称为 resolved(已定型)。

TIP

一般会把 resolved 状态指定为 fulfilled， 不包含 rejected

  • Promise 实例生成后， 可以通过 then 方法指定两个回调函数： resolve 和 reject。
