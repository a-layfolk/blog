# 修改侧边栏

侧边栏的配置在官网 (opens new window)中有详细描述， 我这里也只是说一些小技巧。

我认为侧边栏有两种作用：

  • 一篇文章太长， 侧边栏用于标识标题
  • 几篇文章主题相近， 侧边栏用于引导阅读其他文章， 比如目前你看到的 《VuePress 配置小技巧》 系列文章

关于前者， 我每次都是直接用 yaml 配置生成侧边栏。 具体请看自动生成侧边栏 (opens new window)。

关于后者， 使用官网的多个侧边栏 (opens new window)。由于多个侧边栏需要配置的东西比较复杂， 我们可以抽离出一个 sidebar.js 到 config.js 的同级目录。 然后在需要侧边栏的文件夹下再新建 sidebarItem.js 文件。

举例如下：

我此时的目录结构是

│
├─.vuepress
│     config.js
│     nav.js
│     sidebar.js
│  
│
└─record
    └─vuepress
            README.md
            sidebarItem.js
            修改侧边栏.md
            修改导航栏.md
            修改首页.md

首先是 config.js 的内容

module.exports = {
  ...
  themeConfig: {
    ...
    nav: require("./nav"),
    sidebar: require("./sidebar")
    ...
  }
  ...
}

然后是 sidebar.js 的内容

module.exports = {
  '/record/vuepress/': require("../record/vuepress/sidebarItem")
}

最后是 sidebarItem.js 的内容

module.exports = [
  "", // "" 被默认为 README.md
  "修改首页",
  "修改导航栏",
  "修改侧边栏"
]

通过三层 js 文件，我们便可以非常清晰地生成第二种侧边栏。

TIP

值得注意的是， MarkDown 文件如果被命名为中文， 虽然能够正常解析， 但浏览器的控制台会打印警告。 所以最好命名为英文。
