# Lab 2. 编译器认知实验

# 1. 实验目的和内容

实验目的：

  • 了解工业界常用的编译器 GCC 和 LLVM
  • 熟悉编译器的安装和使用过程
  • 了解编译器工作过程中生成的中间文件格式和内容
  • 了解编译器的优化效果

实验内容：

  • 在 Linux 平台上安装和运行编译器 GCC 和 LLVM
  • 编写简单的测试程序，使用编译器编译
  • 观察中间输出结果

# 2. 实验的具体过程和步骤

实验具体过程：

首先编写简单的代码作为测试用例，熟悉 GCC 和 Clang 的基本编译命令。之后使用计卫星老师 GitHub 中的 C 语言代码作为测试用例。在 Ubuntu 20.04 下对这些测试用例进行编译，每完成一个步骤，记录指令和结果。这部分内容主要记录在 “2. 实验的具体过程和步骤”。

接着使用相同的代码，通过 GCC 和 LLVM 分别对测试代码进行 O0、O1、O2、O3 优化，并对比分析。这部分内容主要记录在 “5. GCC 与 LLVM 对比分析”。

实验具体步骤：

  • 对于 GCC 编译器：
    • 查看编译器的版本
    • 使用编译器编译单个文件
    • 使用编译器编译链接多个文件
    • 查看预处理结果
    • 查看语法分析树
    • 查看中间代码生成结果
    • 查看生成的目标代码（汇编代码）
  • 对于 LLVM 编译器
    • 查看编译器的版本
    • 使用编译器编译链接多个文件
    • 查看编译流程和阶段
    • 查看词法分析结果 1
    • 查看词法分析结果 2
    • 查看语义分析结果 1

# 3. GCC 运行结果分析

GCC 的编译流程分为了四个步骤，分别是：（下述命令以 hello.c 为例子）

**预处理：**把 include 的文件拷贝到要编译的源文件中，在调用宏的地方进行宏替换。

$ gcc -E hello.c -o hello.i

具体如何进行宏替换可以看 2.1.4 的截图。

**编译：**将预处理好的代码转换成汇编语言，进行语义语法分析，如果有错则报错。

$ gcc -S hello.i -o hello.s

具体生成的汇编语言如何，可以看 2.1.7 的截图。

**汇编：**把汇编语言代码转换为目标代码（一定格式的机器代码）。

$ gcc -c hello.s -o hello.o

将 hello.o 以二进制打开再转换为十六进制，效果应该与此图片相似：

**链接：**将生成的 OBJ 文件和系统库的 OBJ 文件链接起来，最终生成可执行文件。

$ gcc -o hello.o -o hello

# 4. LLVM 运行结果分析

LLVM 的编译过程分为七个步骤，分别是：

预处理： 执行后可以看到头文件导入以及宏被替换

$ clang -E main.m -o main_precompile.i

**词法分析：**词法分析会把代码切片成一个个 token

$ clang -fmodules -E -Xclang -dump-tokens main.m
# 或
$ clang -fmodules -E -Xclang -dump-tokens main.m > main_precompile.i

**语法分析：**以语法树的结构说明待编译文件的目录，比如会说明 main 函数的开始结束行，会说明自动释放池参数、字符串变量、调用函数的行号等信息。

$ clang -fmodules -fsyntax-only -Xclang -ast-dump main.m

生成中间代码：

$ clang -S -fobjc-arc -emit-llvm main.m -o main.ll

生成汇编：

$ clang -S main.m -o main_compile.s

生成目标文件：

$ clang -fmodules -c main.m -o main_obj.o

# 5. GCC 与 LLVM 对比分析

编写的测试代码如下，此代码是一个递归求值的斐波那契数列。

#include <stdio.h>

int fibonacci(int num){
  int res;
  if(num < 1){
    res = 0;
  } else if (num <= 2){
    res = 1;
  } else {
    res = fibonacci(num-1)+fibonacci(num-2);
  }
  return res;
}
int main(void){
  int n = 40;
  int res = fibonacci(n);
  printf("%d", res);
  return 0;
}

# 5.3 GCC 和 LLVM 对比

虽然我查阅到的资料大多说：LLVM 的速度更快，出错信息可读性更好，代码结构更清晰。但就此测试程序编译优化而言，GCC 编译后的文件执行速度的确比 LLVM 编译后的文件执行速度快。
