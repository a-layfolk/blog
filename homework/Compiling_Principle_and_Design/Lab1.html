# Lab 1. 程序设计语言认知实验

# 1. 实验目的和内容

实验目的:

  • 了解程序设计语言的发展历史
  • 了解高级程序设计语言的特性
  • 了解编译执行和解释执行两种不同的执行方式
  • 了解语言在编程效率和运行效率之间的差异

实验内容:

  • 使用 C/C++、Java、Python 和 Haskell 4 语言实现快速排序功能
  • 对这几种语言实现的编程效率、程序的规模、程序的运行效率进行对比分析

# 2. 实验的具体过程和步骤

实验具体过程：

使用 C/C++、Java、Python 和 Haskell 种语言编写快速排序代码。用 1000 个， 5000 个， 10 000 个，15 000 个和 20 000 个随机数进行测试， 每个量级的测试 5 次，并计算消耗时间的平均值。

此外，C/C++ 和 Java 补测了 30 000 个和 50 000 个随机数， 每个量级测试 5 次， 并计算消耗时间的平均值。

C/C++、Java 和 Python 快排的划分基准是每次将要排序的数组最右端。同时，三者测试时的随机数大小介于 0 - 100，在各自的主函数中生成。消耗时间由代码内部的计时函数计算，仅为函数调用时间，不包括输出时间。

Haskell 所使用的随机数由 Python 代码生成文件，之后被手动黏贴到 ghci 控制台。消耗时间则使用 :set +s 命令查看。

实验具体步骤：

  • 配置实验环境
  • 编写快速排序的伪代码
  • 编写 C/C++、Java、Python 程序
  • 编写 Haskell 程序
  • 测试不同规模的测试集

# 2.2 编写伪代码

此伪代码参考了《算法导论》，为了简单方便， 划分数组的 主元 总是选择数组的末尾。

partition(arr, left, right)
  x = arr[right]
  i = left - 1
  for j = left to right - 1
      if arr[j] <= x
          i = i + 1
          exchange arr[i] with arr[j]
  exchange arr[i + 1] with arr[right]
  return i + 1

quickSort(arr, left, right)
  if left < right
    mid = partition(arr, left, right)
    quickSort(arr, left, mid - 1)
    quickSort(arr, mid, right - 1)

# 2.3 编写 C/C++、Java、Python 程序

这三种语言编写的程序和伪代码逻辑相差不多。这里列举了我认为差别较大的地方：

  • 实现数值交换

# 2.3.3 实现数值交换

C/C++ 中有指针这一概念， 实现数值交换可以使用指针：

void exchange(int *a, int *b) {
  int tmp = *a;
  *a = *b;
  *b = tmp;
}

Java 中没有指针，可以将数组也作为函数的一项参数：

public static void exchange(int[] arr, int a, int b) {
  int tmp;
  tmp = arr[a];
  arr[a] = arr[b];
  arr[b] = tmp;
}

Python 也没有指针，但 Python 语法可以很方便地支持数值交换：

def exchange(a, b):
  return b, a;

# 2.4 编写 Haskell 程序

Haskell 是纯函数式的编程语言， 和 C/C++、Java、Python 有较大差别。实现快速排序也非常简单。下面的代码参考了 《Haskell 趣学指南》

quicksort :: (Ord a) => [a] -> [a]
quicksort [] = []
quicksort (x:xs) =
  let smallerSorted = quicksort [a | a <- xs, a <= x]
      biggerSorted = quicksort [a | a <- xs, a > x]
  in smallerSorted ++ [x] ++ biggerSorted

至于 Haskell 所需要的随机数， 我是使用 Python 代码生成的， Python 代码如下：

import random

filename = "data.txt"

with open(filename, "w") as file_object:
  file_object.write("[")
  for i in range(1, 5000):
    file_object.write(str(random.randint(1, 100)) + ",")
  file_object.write(str(random.randint(1, 100)))
  file_object.write("]")

# 6. 实验心得体会

本次实验耗时将近 5 天， 如此费时的原因有三：

  • C/C++、Java 和 Python 细节不同
  • Haskell 和过往所学的函数有很大不同
  • 数据量较大

比如光是生成随机数，C/C++ 中使用 rand() % 100, 而 Java 中使用 (int)(Math.random() * 100)，在 Python 中又变成 random.randint(1, 100)。这些都需要我仔细考虑。

至于 Haskell， 这门语言相当小众，资料也很难找。 所以目前我仍未学会如何用 Haskell 读写文件， 这也是在实验中我用 ghci 运行代码的缘故。

由于我的快排使用递归完成，当数据量较大时，很可能递归深度过深。比如我在 Java 中就遇到了堆栈溢出的错误 StackOverflowerror 错误。 在 Python 中遇到了递归深度超过限制的错误。

但多门语言相互比较， 也让我对每种语言的理解都加深了。相信这能让我学习编译原理更加容易。
