# lab3 词法分析实验

# 1. 实验目的和内容

实验目的：

  • 熟悉 C 语言的词法规则，了解编译器词法分析器的主要功能和实现技术。
  • 了解 Flex 工作原理和基本思想，学习使用工具自动生成词法分析器
  • 掌握编译器从前端到后端各个模块的工作原理
  • 掌握词法分析模块与其他模块之间的交互过程
  • 学习正则表达式

实验内容：

  • 安装 Flex
  • 学习 Flex 的基本用法
  • 编写预处理器
    • 编写 Flex 的自定义代码部分
    • 编写 Flex 的规则部分
  • 编写词法分析器
    • 编写 Flex 的自定义代码部分
    • 编写 Flex 的规则部分

# 2. 实现的具体过程和步骤

# 2.1 安装 Flex

实验环境为 Ubuntu 20.04.2.0，安装比较简单，直接输入命令：

$ sudo apt install flex

# 2.2 学习 Flex 的基本用法

flex 的 .l 文件由三部分组成：

{definitions}		// 定义部分
%%
{rules}				// 规则部分
%%
{user subroutines}	// 用户自定义代码部分

  • 定义部分格式一般为：

    name definition

    其中，name 是名字，definition 是任意正则表达式。作用便是给用到的正则表达式去一个别名。

  • 规则部分格式一般为：

    pattern {action}

    其中，pattern 是正则表达式，action 是要执行的具体操作——即一段 C 代码。

  • 用户子程序部分：

    这部分代码会被拷贝到 lex.yy.c 中，用户可以把 main 函数写到其中。

# 2.3 编写预处理器

目前预处理器将要实现的功能包括：

  • 去掉单行注释
  • 去掉多行注释
  • 处理多余的空格

# 2.3.1 编写用户自定义代码部分

首先编写 Flex 的自定义代码部分：

int main (int argc, char *argv[]) {
  yyin = fopen("1_scan_test.c", "r");
  yyout = fopen("1_scan_test.pp.c", "w");
  yylex();
  return 0;
}
int yywrap () { // 返回值为 1，只处理单个文件
  return 1;
}

  • yyin 和 yyout 原本指向标准输入输出，一般为键盘和控制台。这里让它们分别指向 1_scan_test.c 文件和 1_scan_test.pp.c 文件。

    后续可以将这两个文件以 argv[i] （i = 1, 2 ...）替换，达到从输入参数指定输入输出文件的效果。

  • yywrap 的返回值大于 0，表示只处理单个文件。

# 2.3.2 编写规则部分

然后编写规则部分：

...
MultilineComment [/][*][^*]*[*]+([^*/][^*]*[*]+)*[/]
Whitespace [" " \t]+
 
%%

"//" {					// 单行注释
  isMeeted = true;
  char c = input ();
  while (c != '\n') {
    c = input ();		// 吃掉单行注释后的所有内容，包括 \n
  }
}
{MultilineComment} {	// 多行注释
  /* Do Nothing */
  isMeeted = true;
  input ();				// 吃掉多行注释结尾的 \n
}
{Whitespace} {
  // 如果没有注释，则多个空白处理为一个空白
  if (isMeeted == false) {
    fprintf(yyout, " ");
  }
  // 如果有注释，则清空注释之后的空白符
  isMeeted = false;
}
...
  • 处理单行注释：遇到 // 之后一直往后读取，直到遇到 \n ，中间所有的内容（包括 \n）均会被丢弃掉
  • 处理多行注释：使用正则表达式匹配，读取多行注释后不会再做任何事情（相当于清除内容），最后再丢弃 \n
  • 处理多个空白符：凡是出现多个空白符均变为单个空白符

需要注意的是，因为处理多个空白符和处理注释时会产生一些小冲突，所以我定义了一个 isMeeted 作为空白符和注释同时出现的标记。

isMeeted 的作用是：

  • 如果没有注释，则按照正常的多个空白符的处理办法
  • 如果有注释，则注释后的所有空白符均被清除

# 2.4 编写词法分析器

目前词法分析器，主要识别的 token 类型有：

单词 token 类型 单词 token 类型
Keyword Identifier
IntegerConstant FloatingConstant
CharacterConstant StringLiteral
与单词表示一致 EOF（表示文件结束） EOF

输出的 token 格式为 [@0,0:2='int',<'int'>,1:0]

  • 以@开头的数字表示 token 的序号
  • 紧接着的 xx:xx 表示 token 文本对应的开始列和结束列
  • “=” 后面给出了这个范围之内 token 的具体文本
  • “<>”之内表示 token 的类型
  • 最后一个数字对 xx:xx 表示起始行和起始列

# 2.4.1 编写用户自定义代码部分

这部分代码和 2.3.1 的代码变化不大，只是输入文件变为了 1_scan_test.pp.c，输出文件变为了 1_scanner_test.tokens。

int main (int argc, char *argv[]) {
  yyin = fopen("1_scan_test.pp.c", "r");
  yyout = fopen("1_scanner_test.tokens", "w");
  yylex();
  return 0;
}

int yywrap () { // 返回值为 1, 只处理单个文件
  return 1;
}

# 2.4.2 编写定义部分

将定义部分分为两部分：

下面是第一部分：

%option yylineno
%{
#include <stdio.h>
int words = 0;		    // 记录 token 的数量
int columns_begin = 0;	// 记录 token 开始时 列 的位置
int columns_end = 0;	// 记录 token 结束时 列 的位置
%}
...
  • yylineno 由 flex 提供，记录了当前的行号，方便输出单词的开始行
  • words 用于记录 token 列的位置，方便输出 token 的序号
  • columns_begin 和 columns_end 记录开始列和结束列

下面是第二部分：

  • 首先匹配关键字，为这些关键字取一个别名 Keyword，这是定义部分

    ...
Keyword	(auto)|(break)|(case)|(char)|(const)|(continue)|(default)|(do)|(double)|(else)|(enum)|(extern)|(float)|(for)|(goto)|(if)|(inline)|(int)|(long)|(register)|(restrict)|(return)|(short)|(signed)|(sizeof)|(static)|(struct)|(switch)|(typedef)|(union)|(unsigned)|(void)|(volatile)|(while)
...

    接下来是规则部分，每匹配一个关键字，token 计数加 1，并把对应的信息输出到文件中：

    ...
{Keyword} {					// 匹配关键字
  words++;
  columns_end = columns_begin + yyleng;
  fprintf(yyout, "[@%d,%d:%d='%s',<'Keyword'>,%d:%d]\n", words, columns_begin, columns_end, yytext, yylineno, columns_begin);
  columns_begin = columns_end;
}
...

  • 然后是匹配标识符，C 语言标识符由字母或者下划线开头，之后由字母、数字和下划线组成。给标识符定义别名 Identifier

    ...
Digit		[0-9]
NonDigit	[A-Za-z_]
Identifier	{NonDigit}[A-Za-z0-9_]*
...

    接下来是规则部分，和 KeyWords 部分类似，只是输出的类型变为 Identifier

    {Identifier} {
  ...
  fprintf(yyout, "[@%d,%d:%d='%s',<'Identifier'>,%d:%d]\n", words, columns_begin, columns_end, yytext, yylineno, columns_begin);
  ...
}

  • 接着匹配整数，整数包括十进制、八进制和十六进制整数，并且还有 u、U、l、L、ll、LL 等后缀

    ...
NonzeroDigit		[1-9]
DecimalConstant	    {NonzeroDigit}{Digit}*

OctalDigit		    [0-7]
OctalConstant		0{OctalDigit}*

HexadecimalDigit	[0-9a-fA-F]
HexadecimalPrefix	(0x)|(0X)
HexdecimalConstant	{HexadecimalPrefix}{HexadecimalDigit}*

UnsignedSuffix		(u)|(U)
LongSuffix		    (l)|(L)
LongLongSuffix		(ll)|(LL)
IntergerSuffix		{UnsignedSuffix}|{LongSuffix}|{LongLongSuffix}

IntegerConstant	({DecimalConstant}|{OctalConstant}|{HexdecimalConstant}){IntergerSuffix}?
...

    接下来是规则部分，和 Keyword 类似，只是输出的类型变为了 IntegerConstant

  • 紧接着匹配浮点数，浮点数包括十进制和十六进制，且可以采用科学计数法，十进制是科学计数法的标志是 e 或者 E，十六进制时科学计数法的标志是 p 或者 P

    ...
Sign			    (\+)|(\-)
ExponentPart		(e|E){Sign}{Digit}+
FractionalConstant	{Digit}+(\.){Digit}+

BinaryExponentPart	(p|P){Sign}{Digit}+
HexadecimalFractionalConstant	{HexadecimalDigit}+(\.){HexadecimalDigit}+

FloatingSuffix		(f)|(l)|(F)|(L)

DecimalFloatingConstant	({FractionalConstant}{ExponentPart}){FloatingSuffix}?
HexadecimalFloatingConstant	{HexadecimalPrefix}{HexadecimalFractionalConstant}{BinaryExponentPart}{FloatingSuffix}?

FloatingConstant	{DecimalFloatingConstant}|{HexadecimalFloatingConstant}
...

    接下来是规则部分，和 Keyword 类似，只是输出的类型变为了 FloatingConstant

  • 之后是匹配字符，字符包括普通字符比如 a、b 还有特殊字符，比如 \n 、\t 等。且字符由 L 、u、U 等前缀

    ...
CommonCharacters	(\').(\')
SpecialCharacters	(\')(\\).(\')
CharacterPrefix	(L)|(u)|(U)
CharacterConstant	{CharacterPrefix}?{CommonCharacters}|{SpecialCharacters}
...

    接下来是规则部分，和 Keyword 类似，只是输出类型变为了 CharacterConstant

  • 然后是匹配字符串，字符串有着 L、u、U 和 u8 等前缀。

    CommonString		\"(\\.|[^"\\])*\"
StringPrefix		(L)|(u)|(U)|(u8)
StringLiteral		{CommonString}

    接下来是规则部分，和 Keyword 类似，只是输出类型变为了 StringLiteral

  • 对于文件结束 <<EOF>>，需要特殊处理，使其返回值为 0，以免重复执行。

    <<EOF>> {
  words++;
  columns_end = columns_begin + yyleng;
  fprintf(yyout, "[@%d,%d:%d='<EOF>',<EOF>,%d:%d]\n", words, columns_begin, columns_end, yylineno, columns_begin);
  columns_begin = columns_end;
  return 0;
}

  • 至于没有匹配到的符号，基本都是运算符和界限符，则保持原样输出

    . {					// 之前没有匹配的符号，则保持原本的样子
  words++;
  columns_end = columns_begin + yyleng;
  fprintf(yyout, "[@%d,%d:%d='%s',<'%s'>,%d:%d]\n", words, columns_begin, columns_end, yytext, yytext, yylineno, columns_begin);
  columns_begin = columns_end;
}

  • 此外为了准确记录列号，每次换行时，都需要将 columns_begin 和 columns_end 清零

    \n {					// 换行，每次换行之后则重新计算 列 的位置
  columns_begin = 0;
  columns_end = 0;
}

# 4. 实验心得体会

这次的词法分析实验，因为实现方式众多，我也尝试了好几种方式，最终采用 Flex 来完成实验。和其他方式相比，Flex 更加方便快捷，修改也很方便。

在实验过程中，我更加熟悉了正则表达式的书写，深感正则表达式简明而不简单。同样的单词可以用多种正则式来识别，不同正则式之间差别很大，可读性的相差也很大。

当然，在书写正则表达式时，我不可避免地对一些边界情况没有考虑清楚，这也导致我的词法分析器虽然能够识别大部分单词，但偶尔会有一些遗漏之处。这些都需要我之后再一一完善。

备注：本实验在虚拟机的 Ubuntu20.04.2.0 完成，由于我虚拟机出了些问题，能够复制粘贴文本却无法复制粘贴文件，所以 bin 目录下的 exe 文件是在 Windows 10 下重新编译 pp.lex.yy.c 文件和 scannner.lex.yy.c 文件得到的。
