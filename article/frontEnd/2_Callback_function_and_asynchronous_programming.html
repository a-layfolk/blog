# 回调函数与异步

在回调函数与异步这部分， 我比较薄弱。 翻阅部分答案后我有了体会。

实现异步的不是回调函数， 而是其他基础设施 (opens new window)

比如下面的例子， 虽然用了嵌套了几个回调函数， 都只是同步回调， 而不是异步回调。

function func1(func2) {
  func2();
  console.log("1");
}

function func2(func3) {
  func3();
  console.log("2");
}

function func3() {
  console.log("3");
}

func1(function () {
  func2(function () {
    func3();
  });
});
// 输出结果为 3 2 1

而下面这个例子是异步，但这里的异步是由 setTimeout 提供的。 只是实现异步之后， 调用了回调函数而已。

console.log("1");

setTimeout(function() {
  console.log("2");
}, 1000);

console.log("3");
// 输出结果为 1 3 2

TIP

onclick 由浏览器内核的 DOM Binding 模块来处理

setTimeout 由浏览器内核的 timer 模块来处理

ajax 由浏览器内核的 network 模块来处理
